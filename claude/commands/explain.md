Provide a detailed explanation of the specified code or concept.

## Explanation Structure

### 1. High-Level Overview
What does this code/concept do? Explain in plain English first.

### 2. Step-by-Step Breakdown
Walk through the logic piece by piece:
- What each section does
- Why it's structured this way
- How data flows through it

### 3. Key Concepts
Explain any important concepts, patterns, or techniques used:
- Design patterns involved
- Language-specific idioms
- Algorithmic approaches

### 4. Context & Purpose
- Why would someone write this?
- What problem does it solve?
- How does it fit into the larger system?

### 5. Trade-offs & Alternatives
- What are the pros/cons of this approach?
- What are alternative ways to achieve the same thing?
- When would you choose differently?

### 6. Gotchas & Edge Cases
- What could go wrong?
- What assumptions does this code make?
- Edge cases to be aware of

## Style
- Be thorough but organized
- Use analogies if helpful
- Include small code examples to illustrate points
- Assume I want to truly understand, not just get a surface answer
